# ダッシュボード用サマリーデータ取得API

## 前提条件
- AIナレッジ共有サイトのWebアプリで使用するバックエンドのAPI開発を進行中
- ログインAPI、トークン更新API、ユーザー情報取得APIなどがすでに実装済み
- 要件定義書と基本設計書の内容を考慮して、作業手順の検討と実装を行う。

## 実装作業の内容
- GET /api/v1/dashboard エンドポイントの実装
- ログインAPI、トークン更新API、ユーザー情報取得APIなど開発済みの資産があるので、利用可能なファイルなどは活用して実装を進めること。
- 認証やユーザー情報取得とは別のコントローラ、サービス、DTOを新規に作成して実装を行うこと。
- DBのテーブル定義が新規に必要な場合はそちらも追加する。
- テストは単体テストまで実施する。
- すべてのPGの実装が完了してから単体テストの実装を開始する。
- 単体テストの動作確認はdockerコンテナ内で実行する。
- 単体テストは実装済みの単体テストの動作確認も一緒に実行する。
- dockerコンテナの立ち上げと、テストの実行は手動で行う。
- 実装とテストの実行まですべて完了したら、ディレクトリ構成のファイルを更新する。
（NaviAI-Back\docs\nishizawa\NaviAI-Back_ディレクトリ構成.md）

## 手順書に記載する内容
- 概要
- 前提条件
- 参考資料、参考にする要件定義書と基本設計書のファイルパス
- 成果物（想定ファイルやクラス名など）
- 実装方針
- 詳細な実装手順
- チェックリスト
