## 変更概要

「ダッシュボード画面用API」として、以下の2つのエンドポイントを中心に実装されています。
- `GET /api/v1/dashboard` — サマリーデータ取得
- `GET /api/v1/dashboard/activity` — 週次アクティビティデータ取得

## レビュー指摘事項

### 🔴 重大（バグ・セキュリティリスク）

**1. JWTフィルターの二重登録**
SecurityConfig.java にて、既存の `JwtAuthenticationFilter`（`auth/filter/` パッケージ）に加え、新たに `JwtAuthFilter`（`security/` パッケージ）も同じ位置に追加されています。同一リクエストに対してJWT検証が2回走り、`SecurityContext` の上書きや予期しない挙動の原因になります。2つのフィルターの役割を整理し、どちらか一方に統一すべきです。

**2. `JwtAuthFilter` のコンストラクタが二重定義**
JwtAuthFilter.java では `@Autowired(required=false)` のフィールドインジェクションと、同じく `@Autowired(required=false)` のコンストラクタインジェクションが両方定義されています。Springの動作が不定になるため、どちらか一方（コンストラクタインジェクション推奨）に統一してください。

**3. docker-compose.test.yml への `user: root` 追加**
docker-compose.test.yml にテスト用コンテナで `user: root` を追加しています。セキュリティ上の理由で避けるべき設定です。パーミッション問題の根本原因（ファイルの所有権など）を解決する方法を検討してください。

---

### 🟡 中程度（設計・品質）

**4. `DashboardServiceImpl` がネイティブSQLをハードコード**
DashboardServiceImpl.java の `getActivity()` メソッドでは `comment` テーブルや `"like"` テーブルへのSQLが直書きされています。`KnowledgeRepository` に相当するリポジトリが存在しないため、一貫性が低い状態です。`CommentRepository` や `LikeRepository` を作成してJPA経由にするか、SQLを定数や専用クラスに分離してください。

**5. `getSummary()` の週次アクティビティが4回のN+1クエリになっている**
DashboardServiceImpl.java の以下のループは4週分を個別にカウントクエリで取得しており、N+1問題です。
```java
for (int i = weeks - 1; i >= 0; i--) {
    long count = knowledgeRepository.countByCreatedAtBetweenAndDeletedFalse(start, end);
```
`@Query` で1回のGROUP BYクエリに置き換えるべきです。

**6. `new` による `DashboardSummaryResponse.WeeklyActivity` の直接生成**
DashboardSummaryResponse.java の内部クラス `WeeklyActivity` は `@Builder` が付いていないため、`ArticleSummary` と書き方が不統一です。一貫性のためにすべての内部DTOに `@Builder` を付与してください。

**7. `activity` エンドポイントのアノテーションが冗長**
DashboardController.java の `getActivity()` メソッドでは `@RequestParam` と `@DateTimeFormat` が完全修飾名（FQCN）で書かれており、可読性が低いです。通常のimport文を使うべきです。

---

### 🟢 軽微（コードスタイル・改善提案）

**8. `likeCount` が新着記事では常に `0` で返る**
DashboardServiceImpl.java のコメントに「簡略化のため新着一覧では0」と記載があります。フロントエンド側でこの仕様が明確に伝わるよう、API仕様書またはコードのJavadocに明示することを推奨します。

**9. `RbacAspect` の `@RequireRoles` と `@RequirePermissions` の評価ロジック差異**
RbacAspect.java では、`checkRoles` は `anyMatch`（OR）、`checkPermissions` は `allMatch`（AND）と評価方法が異なっています。これが意図的な仕様なのか、コメントで明示してください。

**10. テストクラスの重複**
`DashboardServiceTest` と `DashboardServiceImplTest` が別々のファイルとして存在し、内容が重複しています。どちらか一方に統合してください。
