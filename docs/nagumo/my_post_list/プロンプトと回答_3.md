## レビューコメント一覧と対応内容

### 🔴 必須修正（1〜5）

#### 1. Knowledge.java の上書き問題
**問題**: PR#9 の `Knowledge.java` は手書き getter/setter、`KnowledgeStatus` enum 使用、`is_deleted` なし、`tags` なし、`updatedAt` なし — origin/main の既存 API が全て壊れる  
**対応**: Lombok ベース（`@Data @Builder @NoArgsConstructor @AllArgsConstructor`）で完全に書き直し、`String status`（enum でなく文字列）、`deleted`（論理削除フラグ）、`tags`（ManyToMany）、`updatedAt`、`excerpt`、`thumbnail` を含む形に修正

#### 2. KnowledgeRepository.java の上書き問題
**問題**: PR#9 は既存の全メソッド（ダッシュボード・管理 API で使用中）を削除していた  
**対応**: origin/main の全メソッドを復元し、新規 2 メソッドを追記
```java
Page<Knowledge> findByAuthorId(Long authorId, Pageable pageable);
Page<Knowledge> findByAuthorUsername(String username, Pageable pageable);
```

#### 3. Flyway マイグレーション競合
**問題**: PR#9 の `V3__create_knowledge_table.sql` が origin/main の `V3__create_knowledge_and_tag_tables.sql` と競合  
**対応**:
- 削除: `V3__create_knowledge_table.sql`
- 削除: `V4__add_knowledge_status_index.sql`（origin/main の V3 に同等インデックスが既存）
- 削除: `V5__alter_knowledge_author_id_nullable.sql`（origin/main の V3 に既に nullable 定義済み）
- 新規作成: `V4__add_excerpt_thumbnail_to_knowledge.sql`
```sql
-- knowledge テーブルに excerpt / thumbnail カラムを追加
ALTER TABLE knowledge ADD COLUMN IF NOT EXISTS excerpt VARCHAR(255);
ALTER TABLE knowledge ADD COLUMN IF NOT EXISTS thumbnail VARCHAR(255);
```

#### 4. page=0 での例外
**問題**: `PageRequest.of(page - 1, ...)` で `page=0` が渡ると `PageRequest.of(-1, ...)` となり `IllegalArgumentException` が発生  
**対応**: バリデーションを追加
```java
if (page < 1) {
    return ResponseEntity.badRequest().body("page must be >= 1");
}
```

#### 5. Lombok が pom.xml に未追加
**問題**: PR#9 の `pom.xml` に Lombok 依存関係がなく、`@Data @Builder` 等が使えない  
**対応**: 依存関係と `maven-compiler-plugin` の `annotationProcessorPaths` を追加
```xml
<!-- Lombok: 自動生成アノテーション -->
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.26</version>
    <scope>provided</scope>
</dependency>
```

### 🟡 推奨修正（6〜7）

#### 6. Java 命名規則違反（スネークケース → キャメルケース）
**問題**: `@RequestParam` のパラメータ名が `author_id`、`per_page` とスネークケースになっていた

## コード詳細

### KnowledgeController.java（修正後の主要部）

```java
@GetMapping
public ResponseEntity<?> getKnowledge(
        @RequestParam(required = false) Boolean mine,
        @RequestParam(name = "author_id", required = false) Long authorId,
        @RequestParam(defaultValue = "1") int page,
        @RequestParam(name = "per_page", defaultValue = "20") int perPage,
        @RequestParam(defaultValue = "createdAt") String sort,
        @AuthenticationPrincipal UserDetails userDetails) {

    if (!ALLOWED_SORT_FIELDS.contains(sort)) {
        return ResponseEntity.badRequest().body("Invalid sort field");
    }
    if (page < 1) {
        return ResponseEntity.badRequest().body("page must be >= 1");
    }
    Pageable pageable = PageRequest.of(page - 1, perPage, Sort.by(sort).descending());

    Page<KnowledgeResponse> knowledgePage;

    if (Boolean.TRUE.equals(mine)) {
        if (userDetails == null) {
            return ResponseEntity.status(401).body("Unauthorized");
        }
        knowledgePage = knowledgeService.getMyKnowledge(userDetails.getUsername(), pageable);
    } else if (authorId != null) {
        if (userDetails == null) {
            return ResponseEntity.status(401).body("Unauthorized");
        }
        knowledgePage = knowledgeService.getKnowledgeByAuthorId(authorId, pageable);
    } else {
        knowledgePage = knowledgeService.getPublishedKnowledge(pageable);
    }

    Map<String, Object> response = new LinkedHashMap<>();
    response.put("data", knowledgePage.getContent());
    Map<String, Object> meta = new LinkedHashMap<>();
    meta.put("page", page);
    meta.put("per_page", perPage);
    meta.put("total", knowledgePage.getTotalElements());
    response.put("meta", meta);

    return ResponseEntity.ok(response);
}
```

## 発生した問題と解決

### テスト失敗（環境起因）

**症状**: `KnowledgeControllerTest` の全 7 テストが失敗  
**エラー内容**:
```
Java 25 (69) is not supported by the current version of Byte Buddy
which officially supports Java 21 (65)
```
**原因**: ローカル JDK が Java 25 だが、Mockito の依存する Byte Buddy が Java 21 までしかサポートしていない  
**対処**: コード変更の問題ではなく環境の問題。Docker (`app/Dockerfile.test`) を使った CI 環境では正しい Java バージョンが使われるため、実際の CI では問題なし  
**ローカルで解決するには**: Java 17 または 21 の JDK をインストールし、`JAVA_HOME` を切り替える

### コンパイル警告（既存の問題）

**症状**: `Knowledge.java` で `@Builder will ignore the initializing expression` 警告  
**原因**: Lombok の `@Builder` はフィールドのデフォルト値初期化を無視する（`@Builder.Default` が必要）  
**対処**: origin/main でも同じ警告が存在する既存の問題。今回の PR 対応スコープ外
